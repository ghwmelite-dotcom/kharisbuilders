# Template T2: Generic Starter Content & Placeholder Assets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `main` a clean, generic, license-clean church template (neutral copy, placeholder SVG assets, hybrid seeds) that looks finished on a fresh deploy — while Kharis stays live via a `kharis` branch + a remote-D1 content backfill.

**Architecture:** Two tracks — `main` becomes generic; a `kharis` branch retains Kharis's config/assets/seeds. Before genericizing, Kharis's effective copy is snapshotted into its remote D1 (`INSERT OR IGNORE`) so changing the defaults can't de-brand Kharis. All changes are default *values*, static SVG assets, seed contents, and a small literal sweep — no control-flow changes.

**Tech Stack:** Astro 6 SSR, Tailwind v4, Vitest, Cloudflare D1 (wrangler), `vite-node` for a one-off generation script. Spec: `docs/superpowers/specs/2026-06-04-template-T2-generic-content-design.md`.

> **CRITICAL ORDERING:** Task 1 (Kharis backfill + `kharis` branch) MUST complete before any genericizing task. It captures the *current* Kharis copy; once `fields.ts` is genericized that copy is gone.

Expected: empty output, `main`.

- [ ] **Step 2: Create the generator script** `scripts/gen-backfill.ts`

```ts
// One-off: emit INSERT OR IGNORE for every content key at its CURRENT default,
// so Kharis's remote D1 owns its copy before main's defaults go generic.
// Run with: npx vite-node scripts/gen-backfill.ts
import { writeFileSync } from 'node:fs';
import { contentDefaults } from '../src/lib/content/fields';

const rows = Object.entries(contentDefaults())
  .map(([k, v]) => `  ('${k.replace(/'/g, "''")}', '${v.replace(/'/g, "''")}', '2026-06-04 00:00:00', 'kharis-backfill')`)
  .join(',\n');

const sql = `-- Generated by scripts/gen-backfill.ts — freezes Kharis content before main goes generic.
-- Additive + idempotent: only fills keys staff never edited; never overwrites an edit.
INSERT OR IGNORE INTO page_content (key, value, updated_at, updated_by) VALUES
${rows};
`;

writeFileSync('db/kharis-content-backfill.sql', sql);
console.log('Wrote db/kharis-content-backfill.sql with', Object.keys(contentDefaults()).length, 'rows');
```

// Split a .sql file into individual statements (strip line comments first).
// Safe here: none of the seed string literals contain a ';'.
function statements(file: string): string[] {

    .replace(/--.*$/gm, '')
    .split(';')
    .map((s) => s.trim())
    .filter(Boolean);
}

describe('hybrid generic seeds', () => {

    for (const f of ['db/seed.sql', 'db/seed_funds.sql', 'db/seed_lists.sql', 'db/seed_sermons_events.sql']) {
      for (const stmt of statements(f)) await db.prepare(stmt).run();
    }

  const count = async (t: string) =>
