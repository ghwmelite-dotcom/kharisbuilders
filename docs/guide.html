For church staff

Run the website day-to-day

Everything lives at your-site/admin. Sign in, then use the sidebar. Tap a task to see how — full click-by-click steps are in the Admin guide.

Open Content, pick a page, edit the boxes, hit Save. To restore the original wording, just clear the box and save — you can’t break a page by emptying a field.
On the same Content pages, image slots have a Choose file button. Upload a JPG/PNG up to 6 MB and save. Saving text never deletes a photo, and vice-versa.
Paste the YouTube or Vimeo link — the site embeds the player. Optionally paste a transcript (it powers the AI study guide & “Ask the Pastor”). Tick Published to show it.
Set the date, time and location. Turn on registration with a capacity if people should sign up — the form stops when it’s full. You can export the sign-up list as a CSV.
Create the Funds people can give to (Tithe, Offering, Building…), see every donation under Giving, and manage recurring gifts under Recurring. (Giving must be switched on first — see “Operate & go live”.)
Set the stream link and an Auto / Live / Off switch. When offline, the page shows a countdown to the next gathering and your latest sermon.
In Settings. These appear in the footer and on the Visit page. Also home to leadership, the story timeline, and home-page cards (each its own simple list).
Three golden rules.  1) Changes go live the moment you click Save.  2) Sermons, events & ministries only show publicly when marked Published.  3) Empty a text box + save to bring back the original wording.
For whoever keeps it online

Operate & go live

The site runs immediately. A few services stay dormant until you switch them on — nothing breaks until you do.

Update the live site

New features land on main; your live branch keeps your church’s branding automatically.

# one-time per computer
git config merge.ours.driver true

git checkout kharis        # your live branch
git merge main             # pull in the latest improvements
npm run build
npx wrangler deploy        # push it live

Going-live checklist

Each is a quick, one-sitting task. Tick them off — your progress is saved on this device.

Online giving — Paystack
Create an account → keys → wrangler secret put PAYSTACK_SECRET_KEY → in Settings set paystack_public_key, currency, giving_enabled=true → set the webhook to /api/webhooks/paystack.
Spam protection — Turnstile
Make a widget in Cloudflare → wrangler secret put TURNSTILE_SECRET_KEY → in Settings set the matching turnstile_site_key.
Staff email alerts — Resend
Account + verified domain + key → wrangler secret put RESEND_API_KEY (plus STAFF_EMAIL, FROM_EMAIL).
Lock the admin — Cloudflare Access
In Zero Trust → Access, allow each staff member’s email over /admin + /api/admin.
Smart sermon search — Reindex
Admin → Sermons → Reindex AI search (run once after adding lots of sermons). Until then, search falls back to simple keyword matching.

For developers

Launch a brand-new church

The whole thing is a re-skinnable template — one config file turns it into any church’s site.

Clone & install

Clone the project and run npm install.

Fill in one file

Copy the example config and edit it — name, slug, web address, brand colours, currency, timezone, and which features to enable.

Run one script

It writes the config, names every cloud resource, re-tints the placeholder images to the new brand, and generates a PROVISIONING.md with the exact next commands.

Follow PROVISIONING.md

It lists everything to create the database, storage, and search index, set secrets, lock the admin, and deploy — adapting to the features you enabled.

Commit & deploy

A fully working church site in your brand. Full detail lives in the README.

cp scripts/new-church.config.example.json scripts/new-church.config.json
# edit it, then:
node scripts/new-church.mjs
# then follow the generated PROVISIONING.md

The payments provider that powers online giving.

The command-line tool that deploys the site to Cloudflare.

A toggle — content only shows publicly when it’s marked Published.
tapAstro
The web framework the site is built with — fast pages, modern tooling.
Help, something’s off

Quick fixes

I changed something but don’t see it
Did you click Save? Is the item Published? Refresh the page.
I want the original wording back
Clear the text box and Save.
A photo won’t upload
Must be under 6 MB and a JPG / PNG / WebP.
A photo shows broken or blank
Re-upload it in Content — the stored path may be off.
The website name or logo is wrong
Set once in the config — ask whoever runs the technical side.
I’m locked out of the admin
Ask your administrator to approve your email in Cloudflare Access.
Giving says “coming soon”
Paystack isn’t switched on yet — see Operate & go live.
Sermon search seems basic
Run Admin → Sermons → Reindex AI search once.
